# 로그인 페이지 만들기

## 1. 홈 : 회원들의 이름과 이메일을 표로 표시 -> 로그인한 사용자가 있으면 표시하기

## 2. 홈에서 로그인 버튼 클릭시 로그인 페이지로 이동 -> 유효성검사, 로그인 인증 확인되면 홈으로 이동 -> 로그인 버튼을 로그아웃 버튼으로 바꾸기

## 3. 홈에서 회원가입 버튼을 누르면 회원가입 페이지로 이동 -> 회원가입이 정상적으로 완료되면 자동 로그인후 홈으로 이동


## 4. fbv, cbv 둘다 사용해보기 폼클래스  쓰고 안쓰고 차이 알아보기

## 5. 유저 모델을 기존에 제공하는 모델을 사용할건지, 커스텀을 할건지 부터 정해야한다. 그래서 2가지 다 만들어 볼생각이다.


## 홈페이지 -> 로그인 클릭-> 로그인페이지 -> 아이디,비번 입력 -> (유효성검사 form클래스or직접 구현)-> 인증 검사(authenticate함수 사용 or 직접 모델 탐색해서 아이디와 비번이 일치 하는지확인) -> 로그인 성공후 홈페이지로

## 홈페이지 -> 회원가입 -> 회원가입 페이지 -> 아이디 비번입력 -> 유효성 검사 -> 저장후 자동 로그인 -> 홈페이지로

## 홈페이지 ->  로그아웃 클릭 -> logout함수 사용 or 직접 세션에서 지우기 -> 홈페이지로


## 결국 최고의 방법

- login뷰 사용, form클래스 사용 , 인증함수(authenticate, login , logout)사용(로그인 뷰 사용하면 이것도 안해도됨.)
